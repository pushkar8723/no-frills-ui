# TextArea

Wraps `HTMLTextAreaElement` in a React component to give additional `label` and `errorText` props.

A `*` is added to the label if the field is required.

<Canvas of={TextAreaStories.Demo} />

```jsx
import { TextArea } from 'no-frills-ui';

// Render
<TextArea />;
```

## Props and Ref

All `props` and `ref` are forwarded to the native `textarea` element.
Following props are added on top of `HTMLTextAreaElement` props.

## Accessibility

The TextArea component is fully compliant with WCAG 2.1 AA guidelines:

- **Label Association**: The label is implicitly associated with the textarea through nesting within the `<label>` element.
- **Error Messaging**: Error text is programmatically linked to the textarea via `aria-describedby`, ensuring screen readers announce validation messages.
- **Invalid State**: The `aria-invalid` attribute is automatically set when `errorText` is provided, alerting assistive technologies to validation errors.
- **Required Fields**: The `aria-required` attribute is added when the textarea is marked as required, providing semantic indication beyond visual asterisks.
- **Keyboard Navigation**: Full keyboard support is maintained through native textarea element behavior.

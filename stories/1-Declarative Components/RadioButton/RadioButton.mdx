# RadioButton

Wraps `HTMLInputElement` of type `radio` in a React component. There is also another component `RadioGroup` which
can be used to wrap `RadioButton` component to give a rounded border and a better experience on focus.

`RadioGroup` component is purely for esthetical purpose.

> **Note:** We don't render the `errorText` prop on the `RadioButton` component as
> `RadioButton` component act as a group. This error text is only use to set the
> custom validity of the `input` element.
> If you need to render the error text then use the `ErrorContainer` component.

```jsx
// Render
<RadioGroup>
    <RadioButton name="type" label="D" />
    <RadioButton name="type" label="W" />
    <RadioButton name="type" label="M" />
    <RadioButton name="type" label="Y" />
</RadioGroup>;
```

## Accessibility

The RadioButton component is fully compliant with WCAG 2.1 AA guidelines:

- **Label Association**: The label is implicitly associated with the radio input through nesting within the `<label>` element.
- **Semantic HTML**: Uses native `<input type="radio">` which provides built-in ARIA semantics and behavior.
- **Keyboard Navigation**: Full keyboard support through native radio button behavior (arrow keys for navigation within a group, Space to select).
- **Focus Indication**: Clear focus styles are provided for keyboard navigation, with enhanced z-index management to prevent overlap in RadioGroup.
- **Disabled State**: Properly handles disabled state with appropriate visual styling and prevents interaction.
- **RadioGroup**: The `RadioGroup` component is purely decorative for visual styling. When grouping related radio buttons with a descriptive label, consider using a `<fieldset>` with `<legend>` at the form level for proper semantic grouping.
